# Aryalinux LFS Builder

Automated [LFS 13.0-systemd](https://www.linuxfromscratch.org/lfs/view/stable-systemd/) build from the local HTML book in `13.0/`.

## Layout

| Path | Role |
|------|------|
| `13.0/` | LFS book HTML (source of truth for parsing) |
| `lfs_parser.py` | Package scripts → `stages/`; copies `static/postlfs/*.sh` → `phases/` |
| `parser_rules.py` | Tarball outliers, command drop/strip lists, per-package rules |
| `static/postlfs/` | Curated BLFS post-LFS scripts (initramfs, GRUB EFI, profile) |
| `lfs-build.sh` | Master builder; prompts for disk layout → `config/build.conf` |
| `lib/lfs-common.sh` | Shared helpers (tarballs, logging, `lfs` user, chroot) |
| `lib/lfs-configure.sh` | Interactive configuration (UUIDs from partition paths) |
| `lib/lfs-disk.sh` | Mount by UUID helpers |
| `config/vars.defaults.sh` | Committed defaults (prompt suggestions, kernel version) |
| `config/build.conf` | Generated per machine (gitignored) — **UUIDs only for disks** |
| `stages/stage1–4/` | Parsed package install scripts |
| `phases/` | Non-package steps; ch.9 + boot 50–55, 99 from `static/postlfs/` |

## Build configuration (`config/build.conf`)

On the first build (or `configure`), the builder prompts in sections (defaults in `config/vars.defaults.sh`):

**Disk** — LFS mount; root, ESP, swap partition paths (converted to UUIDs via `blkid`; scripts never use `/dev/sdX`)

**System identity** — OS name/version/codename (default AryaLinux 2026.06 Lazarus), hostname, domain, `/etc/issue` + `/etc/os-release`

**Locale & time** — `LANG` (default `en_IN.UTF-8`), optional `LC_COLLATE` / `LC_TIME`, timezone (default `Asia/Kolkata`), hardware clock UTC vs local

**Console** — keyboard layout (`KEYMAP`), font (`FONT` in `/etc/vconsole.conf`)

**Groff** — paper size `A4` or `letter` (used at `groff` configure + `/etc/papersize`)

**Users** — normal login name (default `aryalinux`), root & user passwords (default `aryalinux`; stored in `build.conf`, chmod 600)

**Network** — `dhcp`, `static` (IP/gateway/DNS), or `none` (offline image)

**Boot** — GRUB install mode, initramfs toggle

Phase scripts `40-console` through `45-devices`, `44-users`, and the `groff` override read these values from `build.conf` (loaded via `lib/lfs-common.sh`).

```bash
sudo ./lfs-build.sh configure   # re-run prompts and refresh build.conf
LFS_YES=1 sudo ./lfs-build.sh all   # accept all bracketed defaults
```

`config/vars.defaults.sh` holds prompt defaults (e.g. `/dev/sda2`); edit that file to change what Enter selects, not the scripts.

### Why UUIDs and caveats

| Issue | Mitigation |
|--------|------------|
| `/dev/sdX` order changes across boots | UUIDs in `fstab` / `grub.cfg`; host builder mounts by partition path |
| Repartition or reformat changes UUID | Re-run `sudo ./lfs-build.sh configure` |
| Chroot has no host `blkid` for your disk | UUIDs resolved on host and stored in `build.conf` (rsync’d into chroot) |
| Separate `/boot` partition | Not automated yet; kernel/initrd paths in `grub.cfg` assume `/boot` on root FS |
| `search --fs-uuid` vs `root=UUID=` | Both use `ROOT_UUID` from the same `build.conf` |

## Parser behaviour

- **Package names** from HTML filenames + `13.0/wget-list` (`parser_rules.py` for outliers). Scripts set only `PKG_TARBALL`; `PKG_DIR` is derived when extracting (top-level directory inside the archive).
- **Command filtering** in `parser_rules.py`:
  - `BUILD_ENV_DROP_PATTERNS` — optional test suites (`make test`, `make -k check`, `su tester …`, env-prefixed variants), chroot/session noise, etc. Continued lines (`\`) are merged before filtering.
  - `BOOK_ONLY_COMMANDS` — documented deny list for book prose that must not become scripts (e.g. `xorriso`, `grub-mkrescue`, per-line `localedef`, libfoo pkg-mgmt tutorial, `grub-install /dev/sdX`, runtime `timedatectl`/`ldconfig`). Curated replacements live in `overrides/`, `phases/`, and `static/postlfs/`.
  - `DROP_BLOCK_PATTERNS` — whole `<pre>` blocks (heredocs, rescue ISO, libfoo examples).
  - `PACKAGE_COMMAND_RULES` — per-slug tweaks after generic filtering.
- **Hand-written phases** (`phases/30-pkg-mgmt`, `40–45`, etc.) are not regenerated by the parser; post-LFS `50+` / `99` come from `static/postlfs/`.
- **Post-LFS:** edit `static/postlfs/`, then `./lfs_parser.py` (clears phases `50+` / `99` and recopies).

Boot order in chroot: `50-fstab` → `52-kernel` → `53-initramfs` → `54-grub-efi` → `55-grub-boot` → `99-blfs-profile`.

## Sources (differs from LFS book ch.3)

| | LFS book | This builder |
|---|----------|--------------|
| Download location | `$LFS/sources` on the mounted partition | `../sources` on the host (sibling of `base-system/`) |
| After mount | `mkdir` + `chmod a+wt` + wget in place | `mount --bind ../sources $LFS/sources` for chapters 5–6 (same idea as ch.7 `$LFS/dev`) |
| Before chroot | — | `umount` the bind, `mkdir` + `chmod a+wt`, **copy** tarballs onto the partition |

```text
aryalinux/
├── base-system/    ← builder
└── sources/        ← host wget cache (outside $LFS)
```

On the first build command (except `configure`), if `../sources` is missing the builder downloads with `wget-list-systemd`, verifies `md5sums`, and `chmod a+wt` the cache (per the book).

After `$LFS` is mounted, the host cache is **bind-mounted** at `$LFS/sources` for stages 1–2. Before chroot (or `chroot-prep`), the bind is removed and sources are **copied** onto the LFS partition (`mkdir`, `chmod a+wt`, per ch.3). The builder tree lives at `$LFS/sources/builder/` throughout.

Populate `../sources` yourself to skip the download.

## Build

```bash
./lfs_parser.py                  # regenerate stages + postlfs phases
sudo ./lfs-build.sh configure    # optional: set up disks first
sudo ./lfs-build.sh all
```

Other commands: `host`, `stage1`, `stage2`, `chroot-prep` (host: chown + kernfs only), `stage3`, `stage4`, `system`, `sync`.

Inside chroot, `lfs-chroot-build.sh` runs ch.7 dirs/files (phases 23–24) before stage3 packages, then stage4 and system configuration.

### Idempotent scripts

Each package and phase script records its run in `$LFS/sources/.lfs-build-logs/`:

| File | Purpose |
|------|---------|
| `<script>.record` | `status`, `started`, `finished`, `duration_seconds` — skip if `status=success` |
| `<script>.log` | Full stdout/stderr transcript |
| `journal.log` | One-line summary per completion |

Re-run the build safely: finished scripts are skipped automatically. To force a redo, delete that script’s `.record` file (and optionally `.log`).

```bash
rm -f /mnt/lfs/sources/.lfs-build-logs/52-kernel.sh.record
```

## Environment notes

- `HOST_LD` in `~lfs/.bashrc` for cross-toolchain builds.
- Chroot is non-interactive (`lfs_chroot_run`).
- Builder tree is copied to `$LFS/sources/builder/` (includes `config/build.conf`).
